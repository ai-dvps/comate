# Dependency Security Upgrade

## Summary

Upgrade all dependencies flagged by `npm audit` to patched versions, using minimal safe bumps and npm `overrides` for transitive fixes. Verify the changes with lint, client/server/browser tests, the production build, and the full Tauri release build.

---

## Problem Frame

`npm audit --registry=https://registry.npmjs.org/` reports 18 vulnerabilities in the project (1 critical, 9 high, 6 moderate, 2 low). Some advisories affect direct dependencies such as `dompurify`, `ws`, `vite`, and `playwright`; others are transitive, including `axios` under `@larksuiteoapi/node-sdk`, `shell-quote`, and `tar`. Several direct dependencies have no upstream fix yet, so remediation must force safe transitive versions through overrides.

Because Comate bundles a Tauri desktop shell and a Node sidecar, dependency changes must be validated end-to-end. A broken build step, native binding mismatch, or incompatible plugin would only surface after install, so the verification bar includes the full `npm run release` pipeline.

This plan implements the requirements captured in `docs/brainstorms/2026-06-29-dependency-security-upgrade-requirements.md`.

---

## Requirements

This plan addresses every requirement from the origin document:

- R1, R2, R3, R4, R5, R6 — vulnerability remediation and zero audit findings.
- R7, R8, R9, R10, R11, R12 — lint, tests, build, and release verification.
- R13, R14 — CHANGELOG update and residual-risk documentation.

---

## Key Technical Decisions

- **Use npm `overrides` for transitive fixes.** Force patched transitive versions from root `package.json` rather than promoting packages like `axios`, `shell-quote`, or `tar` to direct dependencies. This keeps the dependency graph honest and makes overrides easy to remove once upstream packages publish their own fixes.
- **Update transitive sources rather than downgrading direct packages.** `exceljs` is flagged only via `uuid`, and `@yao-pkg/pkg` only via `esbuild`. The plan updates `uuid` and `esbuild` to patched versions instead of downgrading `exceljs` or `pkg`, preserving existing functionality.
- **Accept a Vite major-version bump if required.** The advisory range (`<=6.4.2`) leaves no patched Vite 5.x. If `npm audit` still flags Vite after other updates, bump it to the smallest patched 6.x and update `@vitejs/plugin-react` to a compatible major.
- **Keep non-vulnerable dependencies untouched.** Only packages reported by `npm audit` are upgraded; no opportunistic updates to unrelated dependencies.

## Implementation Units

### U1. Update direct runtime dependencies

**Goal:** Bump `dompurify`, `ws`, and `uuid` to the smallest versions that clear their advisories.

**Requirements:** R1, R2, R4, R6.

**Dependencies:** None.

**Files:**
- `package.json`
- `package-lock.json` (regenerated by `npm install`)

**Approach:**
- Update `dompurify` to a version later than 3.4.10 (e.g., `^3.4.11`) to clear the XSS and hook-pollution advisories.
- Update `ws` to `^8.21.0` or later to clear the memory-exhaustion DoS advisory.
- Update `uuid` to `^11.1.1` or later to clear the buffer-bounds-check advisory. This also resolves the `exceljs` advisory because `exceljs` depends on `uuid`.
- Run `npm install` and confirm `npm audit` no longer flags these packages.

**Patterns to follow:** Use caret ranges consistent with existing declarations.

**Test scenarios:**
- `dompurify`: browser tests pass; manually verify that `sanitizePreviewHtml` in `src/client/lib/sanitize-html.ts` still strips `<script>` tags and keeps allowed markup.
- `ws`: no direct source import exists, but server tests exercise transitive consumers; confirm `npm run test:server` passes.
- `uuid`: server tests create workspaces, sessions, and bots; confirm generated IDs remain valid v4 UUIDs.

**Verification:** `npm audit` no longer reports `dompurify`, `ws`, or `uuid`; `npm run test:server` passes.

### U2. Update build and test dependencies

**Goal:** Bump `esbuild`, `vite`, `playwright`, and companion plugins to patched versions.

**Requirements:** R1, R2, R5, R6.

**Dependencies:** None.

**Files:**
- `package.json`
- `package-lock.json`

**Approach:**
- Update `esbuild` to `^0.28.1` or later to clear the dev-server advisories. This is expected to also clear the `@yao-pkg/pkg` advisory, which is flagged only via `esbuild`.
- Update `playwright` to `^1.55.1` or later to clear the SSL-certificate verification advisory.
- Update `vite` to the smallest patched version that clears its advisories. If Vite 5.x remains flagged, bump to the smallest patched Vite 6.x and update `@vitejs/plugin-react` to a compatible major version.
- Run `npm install` and confirm `npm audit` no longer flags these packages.

**Patterns to follow:** Keep plugin versions aligned with the Vite major line; check `vitest.config.ts` and `vite.config.ts` for any breaking config changes after the bump.

**Test scenarios:**
- `esbuild`: `npm run build:sidecar` bundles the server successfully and the pkg step produces binaries.
- `vite`: `npm run dev:client` starts; `npm run build` completes; Vitest loads both workspace projects.
- `playwright`: `npm run test:browser` executes the Chromium browser tests.

**Verification:** `npm audit` no longer reports `esbuild`, `vite`, or `playwright`; `npm run build` and `npm run test:browser` pass.

### U3. Configure npm overrides for transitive vulnerabilities

**Goal:** Force safe transitive versions for packages with no direct upstream fix.

**Requirements:** R1, R3, R6.

**Dependencies:** U1, U2 (so direct bumps are in place before adding overrides).

**Files:**
- `package.json`

**Approach:**
- Add an `overrides` block at the root of `package.json`.
- Override `axios` to `^1.16.0` or later under both `@larksuiteoapi/node-sdk` and `@larksuite/vercel-chat-adapter` so both nested copies are patched.
- Add root-level overrides for `shell-quote`, `tar`, `form-data`, `hono`, `js-yaml`, `qs`, and `@babel/core` to the smallest patched versions that clear their advisories.
- Run `npm install` so the lockfile reflects the overrides.

**Patterns to follow:** npm `overrides` semantics — nested overrides for packages that need them, root-level overrides for everything else.

**Test scenarios:**
- After install, the nested `axios` copies under `node_modules/@larksuiteoapi/node-sdk/node_modules/axios` and `node_modules/@larksuite/vercel-chat-adapter/node_modules/axios` resolve to a patched version.
- Feishu server tests (`src/server/services/feishu-bot-service.test.ts`, `src/server/services/feishu-user-resolver.test.ts`, etc.) still pass with the overridden `axios`.

**Verification:** `npm audit --registry=https://registry.npmjs.org/` reports zero vulnerabilities.

### U4. Refresh lockfiles and validate zero audit findings

**Goal:** Ensure `package-lock.json` and the workspace lockfile are consistent and the audit is clean.

**Requirements:** R1, R6.

**Dependencies:** U1, U2, U3.

**Files:**
- `package-lock.json`
- `packages/wecom-cli/package-lock.json`

**Approach:**
- Run `npm install` at the root. Because the root declares `workspaces: ["packages/*"]`, this also updates the workspace tree.
- If `packages/wecom-cli/package-lock.json` changes independently, run `npm install -w packages/wecom-cli` as well.
- Run `npm audit --registry=https://registry.npmjs.org/` and confirm zero vulnerabilities across the root and workspace.

**Test scenarios:** None — this is lockfile and audit validation.

**Verification:** `npm audit` exits with zero vulnerabilities and `npm ci` can install from the updated lockfile.

## Scope Boundaries

- **Deferred for later:** proactive upgrades of non-vulnerable dependencies, replacing or removing `@larksuite*` packages if overrides succeed, and setting up automated dependency monitoring or audit CI.
- **Outside this work:** manual runtime testing beyond the automated verification bar, changes to application behavior or features, switching package managers, and dependency policy changes.

---

## Risks & Dependencies

- **Vite major bump may require companion updates.** Mitigation: test `npm run dev:client`, `npm run build`, and Vitest config immediately after bumping Vite; update `@vitejs/plugin-react` to a Vite-6-compatible major if needed.
- **Overridden `axios` may behave differently inside `@larksuite*` packages.** Mitigation: rely on the existing Feishu server test suite; if tests fail, document residual risk and consider pinning a narrower override range rather than the latest `axios`.
- **Native bindings may need rebuilding.** Mitigation: ensure `npm install` rebuilds `better-sqlite3`; verify `scripts/build-sidecar.ts` still finds and copies `better_sqlite3.node` into `src-tauri/resources/`.
- **Playwright browser downloads may fail or be slow.** Mitigation: run `npm run test:browser` in an environment with network access; allow Playwright to fetch its browser binaries if the version bump requires a new Chromium revision.
- **Dependency:** The development environment must be able to run the full Tauri release build, including Rust toolchain and platform-specific signing/config.
